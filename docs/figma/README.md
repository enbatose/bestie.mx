# Bestie.mx Figma user-flow file

This folder contains:

- [`bestie-user-flows.svg`](./bestie-user-flows.svg) — importable **high-level** flow board for Figma.
- [`flows/FLOW-INDEX.md`](./flows/FLOW-INDEX.md) — **screen-by-screen** UI flow specs (routes, layout zones, Figma frame names, components, transitions) so you can edit **individual screens and components** before handoff back to the app.

## How to create the Figma file

1. Open Figma and create a new design file.
2. Drag `docs/figma/bestie-user-flows.svg` onto the canvas, or use **File -> Place image**.
3. Right-click the imported board and choose **Ungroup** if you want to edit individual cards, labels, or arrows.
4. Rename the Figma file to `Bestie.mx - User Flows`.

Figma's native `.fig` file format is not generated by the local toolchain in this repo, so the SVG is the source artifact for creating the editable Figma board.

## Screen-by-screen workflow (recommended)

1. Open [`flows/FLOW-INDEX.md`](./flows/FLOW-INDEX.md) and create **Figma pages** aligned with the table (Shell, Buscar, Publicar, …).
2. For each linked doc, create **frames** using the suggested names (e.g. `Buscar · desktop · map + list`, `Publicar · 2 · Ubicación`).
3. Break each frame into **components** listed under “Components” in that doc; use [`flows/00-figma-conventions.md`](./flows/00-figma-conventions.md) for naming and variants.
4. Connect frames with **prototype arrows** using “Flows out” / “Flows in” sections; keep the SVG board as a **map**, and these frames as the **editable UI source**.

### Real UI from the running app (Figma MCP “code to canvas”)

The first MCP export used **text placeholders** only: Figma’s `use_figma` API can create frames and copy, but it cannot “screenshot” your React tree by itself.

To push the **actual Home (or any) screen** into a Figma file as editable-ish layers:

1. Set `VITE_FIGMA_CAPTURE=1` and run `npm run dev` (see `vite.config.ts` — this injects Figma’s `capture.js` only when that env var is set).
2. In Cursor, use Figma MCP **`generate_figma_design`** with `outputMode: "existingFile"`, your `fileKey`, and optional `nodeId` of the target Figma **page** (e.g. `02 · Home`).
3. Open `http://localhost:5173/` with the **hash URL** returned by the tool (`#figmacapture=…`) so the page submits a capture to Figma.

After capture, Figma adds a frame (often nested) you can select, tweak, and then link back in Cursor for implementation. Turn off `VITE_FIGMA_CAPTURE` for normal dev so production builds never rely on that script.

## Flow coverage

The board is based on the route map in `src/App.tsx`, navigation in `src/components/HeaderMegaMenu.tsx`, and product notes in `docs/PRODUCT_V1.md`.

### 1. Browse and search for a room

- Entry: `/`
- Search: `/buscar`
- Detail: `/anuncio/:id`
- Key states: city chips, URL-driven filters, map pins, results list, listing detail, contact CTA.

### 2. Listing contact and messaging

- Entry: `/anuncio/:id`
- Auth guard: logged-out users open the auth modal.
- Success: `startConversationFromListing` creates a conversation and routes to `/mensajes?c=:conversationId`.
- Alternate contact: listing WhatsApp reveal/link when enabled.

### 3. Authentication and account onboarding

- Modal path: header **Entrar** / **Registro**.
- Full-page paths: `/entrar` and `/registro`.
- Auth methods: WhatsApp OTP, email/password login, email registration.
- Success destination: `/mis-anuncios`.
- Follow-up: profile completion prompt when phone is missing.

### 4. Publish a room or multi-room property

- Entry: `/publicar`.
- Modes: quick room/loft post or property with multiple rooms.
- Steps: property basics, map pin, images, room details, legal confirmation.
- Anonymous guard: autosave draft and redirect to `/entrar`.
- Success: publish and route to `/anuncio/:roomId`.
- Draft success: route to `/mis-anuncios`.

### 5. Manage listings and property lifecycle

- Entry: `/mis-anuncios`.
- Owner actions: edit, upgrade to property, pause, republish, archive.
- Edit route: `/publicar?edit=:propertyId`.
- Statuses: `draft -> published -> paused -> archived`.

### 6. Groups, profile, support, legal, and admin

- Groups: `/grupos`, with a path back to `/publicar`.
- Profile: `/perfil` and `/perfil/editar`.
- Support/legal: `/contacto`, `/faq`, `/legal`, `support@bestie.mx`.
- Admin: `/admin`, shown only for admin users, covering users, cities, analytics, and property/listing moderation actions.
